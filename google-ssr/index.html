手把手搭建Google云及配置SSR教程

· 2022-07-27 · # 笔记

因在工作上需要处理一些国外业务，最近一直在研究谷歌云(Google Cloud Platform)和甲骨文云，在谷歌云搭建的过程中遇到一些坑，下面分享一些经验并做下mark，方便后续使用。

一、提前准备工作

1.有一个谷歌账号

2.能够正常访问google.com

3.已注册Google Cloud Platform

4.已绑定信用卡并获得新注册用户 300 美金的赠送余额。

上述4个准备工作比较简单，按网上一般教程处理即可。

二、实例搭建

1.登陆谷歌云平台的控制面板。

2.打开侧边栏，找到Compute Engine，在弹出的二级菜单中，进入虚拟机实例界面。

3.点击创建实例。

4.名字依自己喜好填，区域我选的洛杉矶，可用区任选，机器配置当然是越便宜好越好，用来科学上学足够了。CPU平台和GPU以及其他选项默认即可！

5.点击窗口的更改选项，右侧的启动磁盘里选择自己擅长的系统，我选择的是CentOS7，不过该系统要求的最小磁盘空间是20G，然后费用就会从原来的5刀+增加到7刀+。

6.身份和API访问权限默认，防火墙的2个选项记得都要勾上，然后点击创建！

7.稍稍等那么一、二分钟，即可在实例列表中看到刚刚建好的实例。这时候实例就完全建立好了

三、配置实例

1.点击刚刚建好的实例中，SSH右边的小箭头，在弹出来的选项中选择点击「在浏览器窗口中打开」

2.稍后出现命令行操作界面：

3.在命令行中输入代码：sudo passwd root为 root 用户设置密码，尽量复杂一点，输完后再输一遍确认！

4.输入代码su root回车，输入密码，使当前用户具备 root 权限。

5.输入代码vim /etc/ssh/sshd_config来编辑 ssh 服务配置文件

6.找到这一行：PermitRootLogin no将默认的「no」改成「yes」

7.找到这一行：PasswordAuthentication no将默认的「no」改成「yes」

8.在这里看要不要顺便把 ssh 端口号（Port）改成高端口（范围 10000~65535），避免密码在默认端口被暴力破解，如果不改的话，按 Esc，输入：:wq确认保存并退出。

9.输入代码/etc/init.d/ssh restart 重新启动 ssh 服务，使更改生效

四、搭建SSR服务器

1.用Xshell连接服务器，输入用户名、密码

2.输入sudo -i先切换到root用户（截图省略）

3.依次运行以下4条命令（截图省略）
　　第1条：yum install -y wget
　　第2条：wget --no-check-certificate https://github.com/teddysun/across/raw/master/bbr.sh
　　第3条：chmod +x bbr.sh
　　第4条：./bbr.sh

4.安装完成后，脚本会提示需要重启VPS，输入 y 并回车后重启, 这时Xshell连接也会自动断开。VPS重启可能需要几分钟, 耐心等待一下。重启完成后，再用Xhell重新连接VPS, sudo -i切换到root用户。

5.验证是否成功安装了最新内核并开启BBR
　　输入命令：uname -r 查看内核版本，如果返回值含有5.15或以上版本, 就表示OK了.
　　输入命令：sysctl net.ipv4.tcp_available_congestion_control 返回值一般为：net.ipv4.tcp_available_congestion_control
　　输入命令：sysctl net.ipv4.tcp_congestion_control 返回值一般为：net.ipv4.tcp_congestion_control = bbr
　　输入命令：sysctl net.core.default_qdisc 返回值一般为：net.core.default_qdisc = fq
　　输入命令：lsmod | grep bbr 返回值有 tcp_bbr 模块即说明bbr已启动。
　　如下图所示：

6.依次运行以下3条命令一键部署SSR服务 (一行一个):
　　第1条：wget --no-check-certificate -O shadowsocks-all.sh https://raw.githubusercontent.com/teddysun/shadowsocks_install/master/shadowsocks-all.sh
　　第2条：chmod +x shadowsocks-all.sh
　　第3条：./shadowsocks-all.sh 2>&1 | tee shadowsocks-all.log

7.出现提示信息, 选择要安装的版本. 推荐选ShadowsocksR, 输入2, 回车
　　设置密码
　　设置端口　　可选范围1-65535, 建议4位以上, 同时避免设置为常用端口22, 80, 443等.
　　选择加密方式 建议选rc4-md5

8.选择协议, 建议选auth_sha1_v4
　选择混淆方式, 选tls1.2_ticket_auth
　然后按任意键开始安装.

9.耐心等待一下, 安装所需时间根据网速, 机器配置等有所区别, 出现类似下图窗口表示安装完成.
　SSR链接出来了，可以尽情的使用了。
